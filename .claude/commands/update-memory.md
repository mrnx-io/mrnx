# Update Memory Bank

Update the Memory Bank to persist context.

## When to Execute

- After implementing features
- After architecture changes
- Before ending session

## Steps

### 1. Update context.md (Always)

- Current Work Focus
- Recent Changes
- Active Components
- Next Steps
- Known Issues

### 2. Update Other Files (If Needed)

- architecture.md - structural changes
- tech.md - stack changes
- tasks.md - new patterns

### 3. Report

```markdown
## Memory Bank Updated

**Files Modified:**
- [x] context.md
- [ ] architecture.md
- [ ] tech.md

**Key Updates:**
- [summary]
```
